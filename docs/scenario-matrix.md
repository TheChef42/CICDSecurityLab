# Scenario Matrix

| Scenario ID | OWASP category | CWE | Scenario title | Vulnerable file(s) | Fixed file(s) | Main weakness | Safe reproduction | Remediation |
|---|---|---|---|---|---|---|---|---|
| SEC03-CWE494 | CICD-SEC-03: Dependency Chain Abuse | CWE-494 | Integrity Omission | `vulnerable/install-unverified.sh` | `fixed/install-verified.sh` | Code is downloaded or executed without checksum, digest, or signature validation. | Run the local script; it executes only a local mock installer. | Verify SHA-256, signatures, or digests before execution. |
| SEC03-CWE829 | CICD-SEC-03: Dependency Chain Abuse | CWE-829 | Untrusted Dependency Resolution | `vulnerable/package.json`, `vulnerable/install-dependencies.sh` | `fixed/package.json`, `fixed/package-lock.json`, `fixed/install-dependencies.sh` | Dependencies are resolved without a lockfile. | Inspect the vulnerable package manifest; no install is required. | Commit a lockfile, pin versions, and use `npm ci`. |
| SEC04-CWE78 | CICD-SEC-04: Poisoned Pipeline Execution | CWE-78 | Tainted Command Injection | `vulnerable/issue-title-search.yml` | `fixed/issue-title-search.yml` | Untrusted GitHub event text is interpolated into shell commands. | Review the workflow file; commands are harmless examples. | Use environment variables, validation, and safe quoting. |
| SEC04-CWE269 | CICD-SEC-04: Poisoned Pipeline Execution | CWE-269 | Token Over-privilege | `vulnerable/build.yml` | `fixed/build.yml` | Workflow token grants unnecessary write access. | Inspect workflow permissions only. | Use least privilege such as `contents: read`. |
| SEC06-CWE798 | CICD-SEC-06: Insufficient Credential Hygiene | CWE-798 | Hardcoded Secret | `vulnerable/deploy.sh`, `vulnerable/app.env` | `fixed/deploy.sh`, `fixed/app.env.example` | Fake credentials are embedded in source files. | Inspect fake test values only. | Read secrets from environment variables or GitHub Secrets. |
| SEC06-CWE532 | CICD-SEC-06: Insufficient Credential Hygiene | CWE-532 | Verbose Log Leak | `vulnerable/debug-workflow.yml` | `fixed/debug-workflow.yml` | Debug logging can print sensitive environment values. | Review local fake values in workflow YAML. | Avoid environment dumps and mask sensitive values. |
| SEC07-CWE16 | CICD-SEC-07: Insecure System Configuration | CWE-16 | Insecure Container Default | `vulnerable/Dockerfile` | `fixed/Dockerfile` | Container runs as root because no `USER` is set. | Build is not required; inspect Dockerfiles. | Create and use a non-root user. |
| SEC07-CWE269 | CICD-SEC-07: Insecure System Configuration | CWE-269 | Overprivileged Cloud Identity | `vulnerable/main.tf` | `fixed/main.tf` | CI/CD IAM policy uses `AdministratorAccess`. | No Terraform apply is required. | Use scoped IAM policy actions. |
| SEC09-CWE345-A | CICD-SEC-09: Improper Artifact Integrity Validation | CWE-345 | Mutable Image Tags | `vulnerable/Dockerfile`, `vulnerable/docker-compose.yml` | `fixed/Dockerfile`, `fixed/docker-compose.yml` | Images are referenced by mutable `:latest` tags. | Inspect image references only. | Pin images by immutable digest. |
| SEC09-CWE345-B | CICD-SEC-09: Improper Artifact Integrity Validation | CWE-345 | Unverified Provenance | `vulnerable/deploy-artifact.sh` | `fixed/deploy-artifact.sh`, `fixed/provenance.json` | Artifact deployment accepts missing or unverified provenance. | Run only against local mock artifacts if desired. | Require provenance or signature verification and fail closed. |

Note: CWE-16 and CWE-345 are broad CWEs. They are used here because the lab examples are configuration and authenticity-verification weaknesses rather than narrow language-specific implementation bugs.
